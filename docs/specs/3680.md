# Spec: Zen Mode — Color Picker via Keyboard Shortcuts

**Issue:** [excalidraw/excalidraw#3680](https://github.com/excalidraw/excalidraw/issues/3680)
**Label:** enhancement
**Complexity:** Low

---

## Goal

Enable color picker popups to appear on screen when triggered by `S` or `G` keyboard shortcuts while Zen Mode is active.

---

## Context

**Issue URL:** https://github.com/excalidraw/excalidraw/issues/3680

**Current behavior:** Pressing `S` (stroke) or `G` (background) with an element selected in Zen Mode sets `appState.openPopup` correctly, but the color picker renders off-screen and is inaccessible.

**Expected behavior:** The color picker popup appears in a visible, usable position on screen.

**Root cause:** Zen Mode applies `transform: translate(-999px, 0)` via the `transition-left` CSS class to `.selected-shape-actions` — the panel that contains the color picker trigger button. Since Radix UI `Popover.Content` is positioned relative to its `Popover.Trigger`, moving the trigger off-screen moves the popup off-screen too.

**Related PRs (both stalled, no reviews):**
- PR #4683 — open, attempted a separate portal container in zen mode (overly complex, abandoned)
- PR #3851 — closed, earlier attempt

## Changes Required

### `packages/excalidraw/components/LayerUI.tsx`

Line ~243, inside `renderSelectedShapeActions()`:

```tsx
// Before
className={clsx("selected-shape-actions zen-mode-transition", {
  "transition-left": appState.zenModeEnabled,
})}

// After
className={clsx("selected-shape-actions zen-mode-transition", {
  "transition-left":
    appState.zenModeEnabled &&
    appState.openPopup !== "elementStroke" &&
    appState.openPopup !== "elementBackground",
})}
```

No other files require changes. The `S`/`G` handler in `App.tsx`, `PropertiesPopover.tsx`, and `ColorPicker.tsx` are all correct as-is.

### `packages/excalidraw/tests/zenMode.test.tsx` (new file)

Add three test cases:
1. Select element → enable Zen Mode → press `S` → assert `openPopup === "elementStroke"` and `.selected-shape-actions` does **not** have class `transition-left`
2. Select element with background → enable Zen Mode → press `G` → assert `openPopup === "elementBackground"` and `.selected-shape-actions` does **not** have class `transition-left`
3. Close picker → assert `.selected-shape-actions` gets `transition-left` back

## Non-Goals

- Showing the color picker without the shape actions panel sliding back in — the issue says "ideally", not required
- Changing the `S`/`G` keyboard handler in `App.tsx` — already works correctly
- Fixing any other popup type (e.g. `elementFill`, `strokeStyle`) in Zen Mode
- Mobile Zen Mode color picker positioning — separate concern
- Modifying `PropertiesPopover.tsx` or `ColorPicker.tsx`

---

## Acceptance Criteria

1. Select an element, enable Zen Mode (`Alt+Z`), press `S` → stroke color picker is visible and interactive on screen.
2. Select an element with background, enable Zen Mode, press `G` → background color picker is visible and interactive on screen.
3. Closing the color picker (Escape or click outside) causes `.selected-shape-actions` to slide back off-screen — Zen Mode is restored.
4. Without a color picker open, Zen Mode still hides the shape panel exactly as before.
5. Opening color pickers via `S`/`G` outside Zen Mode behaves identically to before.
6. `yarn test:update` passes with new test cases covering criteria 1–4.

---

## Edge Cases

- Open stroke picker via `S`, then press `G` without closing — picker switches to background, panel stays visible.
- Open picker via `S` in Zen Mode, then disable Zen Mode (`Alt+Z`) while picker is open — panel and picker remain visible, no visual glitch.
- No element selected in Zen Mode → `S`/`G` do nothing (existing guard in `App.tsx` handles this).
- Element without background selected → `G` does nothing (existing `hasBackground()` guard handles this).

---

## Constraints

- Do not use `position: fixed` or a separate portal container — the existing Radix UI positioning is correct when the trigger is on-screen.
- Change must not affect `transition-right` (top-right UI) or any other zen-mode panel.
- Fewer than 3 files changed (excluding the new test file).
